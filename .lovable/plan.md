

## Law Enforcement Data Request Module

### Overview

A new admin tab called **"LEA Request"** (Law Enforcement Access) where admins can search any user by phone number and generate a structured, downloadable report containing all user data in an official format suitable for government/police authorities.

### Where it lives

- New nav item in the **"Security & Risk"** group: `{ id: "lea_request", label: "LEA Request", icon: Shield }`
- New component: `src/components/admin/AdminLEARequest.tsx`

### How it works

1. **Search by phone number** -- Admin enters a phone number. The system looks up the user in `profiles` table.
2. **Data aggregation** -- Once found, the system fetches:
   - **Profile**: name, phone, email, status, balance, created_at, referral_code
   - **KYC**: full_name, NID number, date_of_birth, KYC status, face_match_result, verified date
   - **Transactions**: all transactions (type, amount, fee, recipient, date, reference, status)
   - **Device registrations**: device fingerprints, registration dates
   - **Login sessions** (if available)
   - **Roles**: any assigned roles
3. **Display in structured format** -- Show all collected data in organized sections on screen
4. **Generate PDF report** -- A "Download Report" button generates a formal PDF (using html2canvas or direct formatting) with:
   - Header: "EasyPay -- Confidential User Data Report"
   - Report ID (auto-generated), date of generation, requesting admin name
   - Fields for "Requesting Authority" and "Reference Number" (admin fills in before download)
   - All user data sections laid out in tables
   - Footer: "This document is confidential and intended solely for the requesting authority"
5. **Audit logging** -- Every search and every download is logged to `audit_logs` with action `lea_data_request`

### Technical details

- PDF generation via `html2canvas` (already in the project) rendering a hidden formatted div, or direct `jspdf` if preferred
- Phone search uses existing `supabase.from("profiles").select("*").eq("phone", phone)`
- Transaction fetch uses `.eq("user_id", userId)` without limit (full history for legal purposes)
- Device fetch from `device_registrations` table
- Admin must fill "Requesting Authority" and "Reference No" fields before download is enabled
- Every action (search + download) inserts an `audit_logs` entry with `action: "lea_data_request"` including the searched phone, authority name, and reference number
